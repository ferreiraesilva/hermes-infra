# Installation Lifecycle

## Draft

Define identity, ownership, environment, permissions and intended products. Validate metadata and record open questions.

## Approval

Obtain human approval for creation, baseline exceptions, deployment changes and required permissions.

## Active

Apply approved baseline contracts, configure project defaults and enable only explicitly approved products.

## Parked

Disable execution while retaining metadata and audit history.

## Retired

Revoke operational permissions, preserve records and mark deployments inactive. Retirement and closure require human acceptance.
