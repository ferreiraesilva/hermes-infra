# New Hermes Installation

Use this flow to create a Hermes Agent installation:

1. Create a unique installation identity.
2. Apply the Hermes Base baseline.
3. Configure GitHub permissions.
4. Configure GitHub Project defaults.
5. Select enabled products from the roadmap product catalog.
6. Register installation metadata in the roadmap installation catalog.
7. Register the product deployment mapping in the roadmap deployment matrix.
8. Request human approval for baseline exceptions or deployment changes.
9. Validate configuration against the Hermes Base schemas.
10. Mark the installation as active only after human approval.

Product catalog membership does not grant deployment. A product must be explicitly approved for the target installation.
